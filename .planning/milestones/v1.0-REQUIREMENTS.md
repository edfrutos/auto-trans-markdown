# Requirements — MarkDown Auto Translator (NOTEBOOK milestone)

**Defined:** 2026-05-28  
**Core value:** Traducir solo texto al usuario sin romper Markdown ni código, con coherencia y coste predecible.

## v1 Requirements

### Hardening (Pre-A)

- [x] **HARD-01**: El sistema rechaza traducciones incompletas cuando el número de segmentos traducidos no coincide con los solicitados
- [x] **HARD-02**: El usuario recibe error claro al subir archivos que no son UTF-8 válido (sin sustitución silenciosa)
- [x] **HARD-03**: La lista de idiomas expuesta refleja las capacidades del proveedor activo (OpenAI vs DeepL)
- [x] **HARD-04**: Existen tests de integración para `translate_segments`, endpoints API y contrato de reassemble

### Pipeline & Core (Phase A)

- [x] **PIPE-01**: Existe `translate_markdown()` en `src/pipeline.py` como única fachada usada por API y CLI
- [x] **GLOS-01**: El usuario puede definir glosario en `glossary.yaml` (no traducir, traducción fija por par de idiomas)
- [x] **GLOS-02**: El usuario puede gestionar entradas del glosario desde la UI web
- [x] **GLOS-03**: Glosario aplicado en editor, archivo único y lote
- [x] **TM-01**: Segmentos traducidos se cachean en SQLite (`data/translation_memory.db`) por hash+idiomas
- [x] **TM-02**: Cache consultada antes de llamar al proveedor; nuevas traducciones se persisten
- [x] **TM-03**: El usuario puede limpiar la memoria de traducción (UI o CLI)
- [x] **CLI-01**: Comando `md-translate file` traduce un `.md` a archivo de salida
- [x] **CLI-02**: Comando `md-translate dir` traduce recursivamente un directorio preservando estructura
- [x] **CLI-03**: Comando `md-translate batch` genera ZIP o directorio de salida
- [x] **CLI-04**: Flag `--dry-run` lista segmentos traducibles sin llamar al proveedor
- [x] **CLI-05**: Entry point `md-translate` apunta a CLI; servidor web tiene comando separado (`serve`)

### Trust & QA (Phase B)

- [x] **VAL-01**: Validador post-traducción comprueba conteo de fences, enlaces e imágenes
- [x] **VAL-02**: Informe de validación visible en UI y opcionalmente en ZIP de lote
- [x] **VAL-03**: Modo `--strict` en CLI bloquea export si validación falla
- [x] **PREV-01**: UI muestra vista previa renderizada del Markdown (original y traducido)
- [x] **PREV-02**: Preview sanitiza HTML (DOMPurify) antes de renderizar
- [x] **PARS-01**: Comentarios traducibles en fences Python (`#`) además de shell
- [x] **PARS-02**: Comentarios traducibles en JS/TS (`//`) y HTML (`<!-- -->`)
- [x] **FM-01**: Frontmatter YAML: traducir campos en lista blanca (`title`, `description`, `summary`)
- [x] **FM-02**: Claves técnicas YAML (`slug`, `date`, `layout`, `id`) nunca se traducen

### Batch UX & Cost (Phase C)

- [x] **JOB-01**: Traducción por lote usa jobs con ID y progreso real vía SSE
- [x] **JOB-02**: UI muestra archivo actual, segmentos y porcentaje durante lote
- [x] **JOB-03**: El usuario puede cancelar un job en curso
- [x] **JOB-04**: Lote devuelve ZIP parcial + `errors.json` si algunos archivos fallan
- [x] **COST-01**: Endpoint `POST /api/translate/estimate` devuelve segmentos, chars y coste estimado
- [x] **COST-02**: UI muestra estimación antes de confirmar traducción grande

### Team Scale (Phase D)

- [x] **MULTI-01**: Usuario puede seleccionar varios idiomas destino en una pasada
- [x] **MULTI-02**: Salida multi-idioma en ZIP con nombres `stem.{lang}.md`
- [x] **DOCKER-01**: `Dockerfile` multi-stage con `requirements.txt` y runtime non-root
- [x] **DOCKER-02**: `docker-compose.yml` con volúmenes para `data/` y `output/`
- [x] **SEC-01**: Variables de entorno para CORS allowlist y bind address documentadas
- [x] **SEC-02**: Límite de tamaño de upload y TTL/limpieza de `output/` configurable

### Editorial & Pro (Phase E)

- [x] **REV-01**: Modo revisión permite editar segmentos traducidos antes de exportar
- [x] **REV-02**: Segmentos marcados como «dudosos» destacados para revisión
- [x] **FALL-01**: Fallback automático DeepL → OpenAI cuando DeepL falla por cuota/idioma
- [x] **DIFF-01**: Vista diff lado a lado original vs traducción (texto traducible resaltado)
- [x] **WATCH-01**: CLI `md-translate watch` monitoriza carpeta y escribe traducciones en salida
- [x] **TREE-01**: CLI traduce árbol de docs respetando `.gitignore` del proyecto fuente
- [x] **TONE-01**: Selector formal/informal para DeepL y tono LLM
- [x] **HIST-01**: Historial de traducciones recientes opt-in (sin API keys ni contenido sensible por defecto)
- [x] **EXPORT-01**: Export opcional a HTML autocontenido desde Markdown traducido

## v2 Requirements (deferred)

- [ ] **V2-01**: Export PDF (WeasyPrint/Pandoc)
- [ ] **V2-02**: Plugin Obsidian o VS Code
- [ ] **V2-03**: Multi-tenant con API key por usuario
- [ ] **V2-04**: Redis job store para despliegue multi-worker

## Out of Scope

| Feature                                   | Reason                                |
| ----------------------------------------- | ------------------------------------- |
| Traducción directa PDF/DOCX               | Pipeline distinto; usar MD intermedio |
| MT offline sin LLM como calidad principal | Inferior en modismos                  |
| Reescritura libre del documento           | Fuera del core value                  |
| Multi-tenant con API keys por usuario     | Diferido a v2 (V2-03)                 |
