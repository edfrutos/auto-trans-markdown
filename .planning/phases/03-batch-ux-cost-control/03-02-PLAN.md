    1. `JobState` enum: pending, running, completed, cancelled.

    2. `BatchJob` dataclass: id (uuid4 str), state, target_lang, source_lang, file_entries list[tuple[str,str]], cancel_requested bool, successes, errors, event_queue asyncio.Queue, zip_bytes optional, task optional.

    3. Module registry `_jobs: dict[str, BatchJob]` + `_lock = asyncio.Lock()`.

    4. `async def _emit(job, event: dict)`: add `"type"` key; `await job.event_queue.put(event)`.

    5. `async def _run_job(job: BatchJob)`:
       - Set state running
       - For index, (filename, content) in enumerate(job.file_entries):
         - If job.cancel_requested: append BatchFileError(filename, "Cancelado: job cancelado por el usuario"); continue OR break after adding remaining as cancelled — break after current file per D-12
         - Emit file_start {filename, file_index, total_files}
         - Define sync progress callback: lambda d,t: asyncio.run_coroutine_threadsafe(_emit(job, {type: segment_progress, filename, done: d, total: t}), loop)
         - try: result = await run_in_executor(translate_markdown, content, TranslateOptions(..., on_progress=callback))
         - except Exception as e: append error, emit error event, continue (D-06)
         - else: append BatchFileSuccess with validation, emit file_done
         - If cancel_requested after file: break
       - job.zip_bytes = build_batch_zip(job.successes, job.errors)
       - state = cancelled if cancel_requested else completed
       - Emit complete {ok_count, error_count, total_files, cancelled: bool}

    6. `create_batch_job(...) -> str`: store job, create asyncio.Task for _run_job, return id.

    7. `cancel_job(job_id) -> bool`: set cancel_requested True.

    8. Tests with mocked translate_markdown:
       - Full success emits complete
       - One file raises → error event + continues
       - cancel after file 1 → state cancelled, zip has partial + errors
  </action>
