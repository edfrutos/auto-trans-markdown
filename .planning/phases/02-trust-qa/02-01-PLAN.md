  <behavior>
    - Test: original y traducido idénticos en estructura → overall pass, todos los checks pass
    - Test: fence extra en traducido → check fences status error, overall error
    - Test: enlace markdown faltante → check links status error
    - Test: imagen `![alt](url)` conteo distinto → check images status error
    - Test: backtick inline count mismatch → check inline_code status error
    - Test: heading depth sequence differs → check headings status error
    - Test: overall error si cualquier check error; warn si solo warns; pass si todos pass
  </behavior>
  <action>
    Per D-03, D-04, RESEARCH.md:

    1. Crear `src/validator.py` con docstring módulo en español y `from __future__ import annotations`.

    2. `@dataclass ValidationCheck`: `id: str`, `status: str` (`pass`|`warn`|`error`), `message: str`, `expected: int | None = None`, `actual: int | None = None`.

    3. `@dataclass ValidationReport`: `overall: str`, `checks: list[ValidationCheck]`.

    4. Constantes check IDs: `fences`, `links`, `images`, `inline_code`, `headings`.

    5. Helper `_strip_fenced_blocks(text: str) -> str` — eliminar contenido entre pares ```/~~~ para checks de headings/inline sin falsos positivos.

    6. Implementar checks (solo stdlib `re`):
       - **fences**: contar líneas apertura ``` o ~~~ (strip-aware); mismatch → `error`
       - **links**: `re.findall(r'(?<!!)\[[^\]]*\]\([^)]+\)', text)`; mismatch → `error`
       - **images**: `re.findall(r'!\[[^\]]*\]\([^)]+\)', text)`; mismatch → `error`
       - **inline_code**: contar runs de backticks (pares); mismatch → `error`
       - **headings**: secuencia profundidad `#` por línea en texto sin fences; cualquier índice difiere → `error`

    7. Reglas v1: desajuste numérico = `error` (no warn en Fase 2 salvo extensión futura). Sin check longitud >300% (deferred D-03).

    8. `validate_translation(original, translated) -> ValidationReport`: ejecutar los 5 checks; `overall = error` si algún error; else `warn` si algún warn; else `pass`.

    9. Mensajes en español orientados al usuario: p. ej. `"Bloques de código: 2 esperados, 3 encontrados"`.

    10. Exportar `validation_to_dict(report: ValidationReport) -> dict` para serialización JSON en planes posteriores.
  </action>
