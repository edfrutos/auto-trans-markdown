  Imports: SwiftUI (no añadir más — Foundation y AppKit se heredan del entorno).

  Struct BatchSheet: View con:
    @Binding var isPresented: Bool
    var manager: BatchJobManager
    var serverManager: ServerManager

  El body usa VStack(alignment: .leading, spacing: 0) con cabecera HStack + Divider + contenido principal, siguiendo el patrón de SettingsView.swift. La cabecera muestra "Traducir lote" como Text(.headline) con Image(systemName: "tray.and.arrow.up.fill") al estilo del proyecto.

  El contenido principal es un switch sobre manager.jobState con tres ramas:

  RAMA 1 — .prepared(let urls):
    - Lista de los nombres de archivo (urls.map { $0.lastPathComponent }) en un ScrollView con altura máxima ~150 pt, cada elemento como Text con .font(.caption) y padding vertical pequeño
    - Text con el idioma destino: "Idioma destino: \(UserDefaults.standard.string(forKey: "defaultTargetLang") ?? "es")"
    - Barra de botones HStack al pie: Spacer() + Button("Cancelar") { isPresented = false } + Button("Traducir") { Task { await manager.start(port: serverManager.serverPort, targetLang: UserDefaults.standard.string(forKey: "defaultTargetLang") ?? "es") } }
    - El botón Traducir lleva .keyboardShortcut(.return) y se deshabilita si serverManager.state != .running (servidor no listo)

  RAMA 2 — .running o .cancelling (usar dos casos con misma vista base):
    Para obtener el texto de estado, detectar si jobState es .cancelling para mostrar "Cancelando — terminando archivo en curso…" en lugar del nombre del archivo en curso.
    - Barra global determinada: ProgressView(value: Double(manager.filesDone), total: Double(max(manager.filesTotal, 1))).progressViewStyle(.linear)
    - Contador: Text("\(manager.filesDone) de \(manager.filesTotal) archivos").font(.caption)
    - Nombre archivo en curso: Text(manager.currentFile).font(.subheadline).lineLimit(1).truncationMode(.middle)
    - Barra de segmentos del archivo: ProgressView(value: Double(manager.segmentsDone), total: Double(max(manager.segmentsTotal, 1))).progressViewStyle(.linear).tint(.secondary)
    - Texto de segmentos: Text(segmentProgressText).font(.caption2).foregroundStyle(.secondary) donde el texto es "Segmento \(manager.segmentsDone)/\(manager.segmentsTotal)" o vacío si segmentsTotal == 0
    - Barra de botones HStack: Spacer() + Button("Continuar en segundo plano") { isPresented = false } + Button("Cancelar") { manager.cancel() }.disabled(isCancelling) donde isCancelling se deriva de si jobState == .cancelling
    - Si jobState == .cancelling, mostrar Text("Cancelando — terminando archivo en curso…").font(.caption).foregroundStyle(.orange) sobre los botones

  RAMA 3 — .done(let ok, let errors, let cancelled):
    - Texto de resumen: si cancelled, "Cancelado: \(ok) de \(manager.totalCount) traducidos"; si !cancelled y errors.isEmpty, "\(ok) archivos traducidos correctamente"; si !cancelled y !errors.isEmpty, "\(ok) traducidos · \(errors.count) errores"
    - Si errors no está vacío: ScrollView con lista de errores como Text("• \(filename): \(msg)").font(.caption).foregroundStyle(.red) por cada (filename, msg) en errors
    - Barra de botones HStack: Spacer() + Button("Cerrar") { manager.reset(); isPresented = false } + Button("Mostrar en Finder") { OutputManager.shared.revealOutputFolder() }

  RAMA DEFAULT (.idle): EmptyView() — la sheet no debería mostrarse en .idle, pero la vista debe ser exhaustiva.

  Al pie de la vista, fuera del switch, añadir un padding(.bottom, 20) y padding(.horizontal, 24) al VStack completo para coherencia con SettingsView.

  La sheet tiene frame mínimo: .frame(minWidth: 480, minHeight: 300). Siguiendo el patrón de SettingsView, NO usar NavigationView ni NavigationStack — la sheet es plana.

  Textos de UI: todos en español de España per convención del proyecto.

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-06 | Information Disclosure | Lista de archivos en .prepared (ruta completa visible en la sheet) | accept | Los nombres de archivo se muestran como lastPathComponent (solo nombre, sin ruta completa) — no se expone la ruta al sistema de archivos en la UI |
| T-18-07 | Denial of Service | ScrollView de errores con lotes muy grandes (cientos de errores) | accept | Los errores se almacenan en errorMessages (array en memoria); en Phase 18 el límite práctico es MAX_BATCH_UPLOAD_MB=50MB del backend; el ScrollView tiene altura limitada — aceptable |
| T-18-08 | Spoofing | Mensajes de error del backend en la sheet (message del evento SSE "error") | mitigate | Los mensajes de error se muestran tal cual del backend (texto plano, no HTML/JS); SwiftUI Text() no interpreta HTML ni ejecuta código — sin riesgo de XSS en entorno local |
| T-18-SC | Tampering | npm/pip/cargo installs | accept | Esta vista no instala paquetes — solo SwiftUI nativo |
