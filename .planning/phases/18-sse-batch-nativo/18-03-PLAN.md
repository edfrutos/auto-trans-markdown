<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| application(_:open:) — URLs del sistema | macOS valida las URLs antes de pasarlas al delegate; el filtro .md ya existe en Phase 13 |
| applicationShouldTerminate — alert modal | Código AppKit síncrono; el alert bloquea la terminación hasta la respuesta del usuario |
| .onReceive(.openBatchSheet) — notification object | La notification puede llevar [URL] como object desde Commands.openBatchFiles; AppDelegate no usa el object (ya llamó prepareWith antes de postear) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-09 | Tampering | .openBatchSheet notification object — cualquier código puede postear esta notificación con URLs arbitrarias | mitigate | El receptor (MDTranslatorApp.onReceive) solo activa showBatchSheet = true; las URLs se leen de BatchJobManager.shared.jobState que ya fue inicializado por el emisor legítimo (AppDelegate o Commands). Un emisor malicioso podría abrir la sheet vacía (prepared([])); el botón Traducir con array vacío produciría un job vacío — el backend lo rechazará con 400. Riesgo bajo en app local |
| T-18-10 | Denial of Service | applicationShouldTerminate — alert bloqueante impide salir | accept | El alert es estándar de macOS (NSAlert.runModal); el usuario siempre puede elegir "Salir y cancelar". No hay bucle ni bloqueo indefinido |
| T-18-11 | Information Disclosure | Logs del servidor (/tmp/md-translator-server.log) visibles en Terminal | accept | Los logs son locales, en /tmp del usuario; contienen rutas y nombres de archivo del lote — datos del propio usuario en su máquina local; sin exposición a terceros |
| T-18-SC | Tampering | npm/pip/cargo installs | accept | Este plan solo modifica AppDelegate.swift y MDTranslatorApp.swift — sin paquetes nuevos |

</threat_model>
