## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App Swift → Backend local (127.0.0.1:{port}) | Toda la superficie HTTP: POST multipart, GET SSE stream, DELETE, GET download |
| ZIP descargado → Sistema de archivos local | Extracción con /usr/bin/unzip sobre archivos controlados por el backend |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-18-01 | Tampering | NSOpenPanel.urls en openBatchFiles() | mitigate | Filtrar URLs a extensión .md (lowercased) antes de pasar a BatchJobManager.prepareWith(); rechazar silenciosamente archivos no-.md |
| T-18-02 | Elevation of Privilege | SSRF — URL de la request SSE | mitigate | URL hardcodeada a "http://127.0.0.1:\(port)/..." donde port viene de ServerManager.serverPort (bind dinámico local); no interpolación de input de usuario |
| T-18-03 | Tampering | ZIP bomb / zip slip en extracción | mitigate | /usr/bin/unzip con flag -j (junk paths — elimina estructura de directorio del ZIP, impide path traversal); solo archivos *.md extraídos; el ZIP lo produce el backend local de confianza |
| T-18-04 | Spoofing | API_TOKEN Bearer si activo | accept | En entorno local de desarrollo el API_TOKEN está vacío (no-op). Si el usuario lo configura, las llamadas fallarán con 401 — documentar en resumen de la fase que el cliente Swift debe añadir el header Bearer si API_TOKEN está configurado en .env. Riesgo bajo: solo acceso local |
| T-18-05 | Denial of Service | Data(contentsOf: url) para archivos .md grandes | accept | Límite backend MAX_BATCH_UPLOAD_MB=50 MB total rechaza el upload; los archivos .md de documentación son raramente > 1 MB; aceptable para Phase 18 |
| T-18-SC | Tampering | npm/pip/cargo installs | accept | Esta fase no instala paquetes externos — Package Legitimacy Audit confirma "ninguna dependencia nueva" en RESEARCH.md |
