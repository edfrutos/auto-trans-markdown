# Plan 10-03 — Keychain: API keys seguras
## Phase: 10 — Swift App Shell & WebView

**Estado:** PLANIFICADO  
**Objetivo:** Almacenar `OPENAI_API_KEY` y `DEEPL_API_KEY` en el Keychain de macOS y pasarlas al subprocess uvicorn vía variables de entorno. Eliminar la dependencia de `.env` en la app bundle. Añadir una pantalla de configuración nativa para la primera ejecución (o cuando no hay keys guardadas).

---

## Contexto

- La decisión de diseño (STATE.md) es: "API keys en Keychain macOS via Security.framework — nunca en `.env` ni argumentos CLI".
- El backend Python lee las keys desde `os.getenv("OPENAI_API_KEY")` etc. — ya hereda el entorno del proceso padre (fix del Plan 09-03). Basta con inyectarlas en `p.environment` antes de `p.run()`.
- Sin App Sandbox, no hay restricciones especiales para acceder al Keychain (el `kSecAttrAccessGroup` por defecto funciona).
- El `.env` actualmente en el workspace NO debe copiarse al bundle (ya está en `.gitignore`). En producción, la app no tendrá `.env`.

---

## Tareas

### 1. `KeychainManager.swift` — wrapper de Security.framework

```swift
// KeychainManager.swift
import Foundation
import Security

enum KeychainManager {
    static let service = "com.edefrutos.md-translator"

    static func save(key: String, value: String) throws {
        let data = Data(value.utf8)
        let query: [CFString: Any] = [
            kSecClass:       kSecClassGenericPassword,
            kSecAttrService: service,
            kSecAttrAccount: key
        ]
        // Intentar actualizar primero
        let status = SecItemUpdate(query as CFDictionary,
                                   [kSecValueData: data] as CFDictionary)
        if status == errSecItemNotFound {
            var addQuery = query
            addQuery[kSecValueData] = data
            let addStatus = SecItemAdd(addQuery as CFDictionary, nil)
            guard addStatus == errSecSuccess else {
                throw KeychainError.saveFailed(addStatus)
            }
        } else if status != errSecSuccess {
            throw KeychainError.saveFailed(status)
        }
    }

    static func load(key: String) -> String? {
        let query: [CFString: Any] = [
            kSecClass:            kSecClassGenericPassword,
            kSecAttrService:      service,
            kSecAttrAccount:      key,
            kSecReturnData:       true,
            kSecMatchLimit:       kSecMatchLimitOne
        ]
        var result: AnyObject?
        let status = SecItemCopyMatching(query as CFDictionary, &result)
        guard status == errSecSuccess,
              let data = result as? Data,
              let value = String(data: data, encoding: .utf8) else { return nil }
        return value
    }

    static func delete(key: String) {
        let query: [CFString: Any] = [
            kSecClass:       kSecClassGenericPassword,
            kSecAttrService: service,
            kSecAttrAccount: key
        ]
        SecItemDelete(query as CFDictionary)
    }

    enum KeychainError: LocalizedError {
        case saveFailed(OSStatus)
        var errorDescription: String? {
            switch self { case .saveFailed(let s): return "Keychain error: \(s)" }
        }
    }
}
```

**Keys a gestionar:**

| Keychain account | Uso |
|-----------------|-----|
| `OPENAI_API_KEY` | Auth OpenAI |
| `DEEPL_API_KEY` | Auth DeepL |
| `TRANSLATION_PROVIDER` | `openai` \| `deepl` (persistir preferencia) |

### 2. `SettingsView.swift` — pantalla de configuración nativa

Mostrar en dos casos:
1. Primera ejecución: ninguna key guardada en Keychain.
2. Manualmente vía menú MD Translator > Configuración… (⌘,).

```swift
// SettingsView.swift
import SwiftUI

struct SettingsView: View {
    @State private var openAIKey = KeychainManager.load(key: "OPENAI_API_KEY") ?? ""
    @State private var deepLKey  = KeychainManager.load(key: "DEEPL_API_KEY") ?? ""
    @State private var provider  = KeychainManager.load(key: "TRANSLATION_PROVIDER") ?? "openai"
    @State private var saved     = false

    var body: some View {
        Form {
            Section("Proveedor") {
                Picker("Proveedor", selection: $provider) {
                    Text("OpenAI").tag("openai")
                    Text("DeepL").tag("deepl")
                }
                .pickerStyle(.segmented)
            }
            Section("API Keys") {
                SecureField("OpenAI API Key", text: $openAIKey)
                SecureField("DeepL API Key", text: $deepLKey)
            }
            HStack {
                Spacer()
                Button("Guardar") { saveKeys() }
                    .buttonStyle(.borderedProminent)
                    .disabled(openAIKey.isEmpty && deepLKey.isEmpty)
            }
        }
        .formStyle(.grouped)
        .padding()
        .frame(width: 440, height: 260)
        .alert("Keys guardadas", isPresented: $saved) {
            Button("OK") {}
        }
    }

    private func saveKeys() {
        try? KeychainManager.save(key: "OPENAI_API_KEY",       value: openAIKey)
        try? KeychainManager.save(key: "DEEPL_API_KEY",        value: deepLKey)
        try? KeychainManager.save(key: "TRANSLATION_PROVIDER", value: provider)
        saved = true
    }
}
```

### 3. Flujo de primera ejecución en `MDTranslatorApp`

```swift
// En MDTranslatorApp.body, antes de iniciar ServerManager.start():
// Si no hay ninguna key → mostrar SettingsView modal

@State private var showSettings = false

// Dentro de .task (en SplashView o en App.body):
let hasKey = KeychainManager.load(key: "OPENAI_API_KEY") != nil
             || KeychainManager.load(key: "DEEPL_API_KEY") != nil
if !hasKey {
    showSettings = true
    // Suspender start() hasta que el usuario guarde al menos una key
    return
}
await serverManager.start()
```

Añadir `.sheet(isPresented: $showSettings) { SettingsView() }`.

### 4. Inyectar keys en `ServerManager.start()`

En el bloque donde se construye `env` (antes de `p.run()`):

```swift
// Inyectar keys desde Keychain
if let openAIKey = KeychainManager.load(key: "OPENAI_API_KEY") {
    env["OPENAI_API_KEY"] = openAIKey
}
if let deepLKey = KeychainManager.load(key: "DEEPL_API_KEY") {
    env["DEEPL_API_KEY"] = deepLKey
}
if let provider = KeychainManager.load(key: "TRANSLATION_PROVIDER") {
    env["TRANSLATION_PROVIDER"] = provider
}
p.environment = env
```

> **Pitfall:** No incluir las keys en `p.arguments` ni loguearlas. El log del servidor (`md-translator-server.log`) no debe contener API keys.

## Criterios de aceptación

- [ ] Primera ejecución sin keys → `SettingsView` aparece antes de intentar arrancar uvicorn.
- [ ] Tras guardar key válida, el servidor arranca y la traducción funciona end-to-end.
- [ ] Las keys son legibles por `KeychainManager.load` en reinicios posteriores (persistencia Keychain).
- [ ] Las keys NO aparecen en `md-translator-server.log` ni en ningún otro log.
- [ ] Menú ⌘, abre `SettingsView` desde cualquier estado de la app.
- [ ] Cambiar de proveedor en settings y reiniciar el servidor refleja el cambio (`TRANSLATION_PROVIDER` nuevo).

## Notas de seguridad

- `SecureField` en SwiftUI enmascara el texto — evitar el campo `.textFieldStyle(.plain)` sin enmascarar.
- El acceso al Keychain en macOS muestra un diálogo de autorización la primera vez en producción (sin App Sandbox, el Keychain se comparte con la cuenta del usuario). En desarrollo con "Sign to Run Locally" esto no ocurre.
- En Phase 12 (distribución), si se usa App Sandbox, habrá que añadir el entitlement `keychain-access-groups` y revisar el `kSecAttrAccessGroup`.
