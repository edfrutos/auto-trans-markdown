# Plan 10-01 — WKWebView: embeber la UI web existente
## Phase: 10 — Swift App Shell & WebView

**Estado:** PLANIFICADO  
**Objetivo:** Reemplazar `Text("Main UI — Phase 10")` por un `WKWebView` que cargue la interfaz FastAPI/HTML existente (`static/index.html`) servida por uvicorn en `http://127.0.0.1:{puerto}`.

---

## Contexto

- `serverManager.serverPort` contiene el puerto dinámico asignado en `ServerManager.start()`.
- El backend sirve el frontend en `/` → `static/index.html` (montaje estático en `src/main.py`).
- La ventana actual tiene `defaultSize(width: 400, height: 220)` — apropiado para la splash, pero la UI web necesita más espacio.
- `WKWebView` es la única opción en macOS para embeber contenido web en una app nativa (SwiftUI no tiene `WebView` nativo).

---

## Tareas

### 1. Añadir `WebView.swift` — `NSViewRepresentable` wrapper

```swift
// WebView.swift
import SwiftUI
import WebKit

struct WebView: NSViewRepresentable {
    let url: URL

    func makeNSView(context: Context) -> WKWebView {
        let config = WKWebViewConfiguration()
        // Permitir acceso a localhost (desactivar la política CORS interna de WebKit para localhost)
        config.preferences.setValue(true, forKey: "developerExtrasEnabled")
        let webView = WKWebView(frame: .zero, configuration: config)
        webView.load(URLRequest(url: url))
        return webView
    }

    func updateNSView(_ nsView: WKWebView, context: Context) {
        // Solo recargar si la URL cambió (raro: el puerto no cambia en caliente)
        if nsView.url?.host != url.host || nsView.url?.port != url.port {
            nsView.load(URLRequest(url: url))
        }
    }
}
```

**Nota:** `developerExtrasEnabled` activa el inspector web vía clic derecho → "Inspect Element". Útil en desarrollo; podría retirarse en distribución pero no perjudica.

### 2. Modificar `MDTranslatorApp.swift` — usar WebView cuando `state == .running`

```swift
// Reemplazar:
Text("Main UI — Phase 10")
    .frame(width: 400, height: 220)

// Por:
WebView(url: URL(string: "http://127.0.0.1:\(serverManager.serverPort)")!)
```

Actualizar también `defaultSize` y los `frame` de la splash a las dimensiones objetivo de la UI web.

### 3. Dimensiones de la ventana

| Estado | Width | Height |
|--------|-------|--------|
| Splash (idle/starting/failed) | 400 | 220 |
| UI web (running) | 1 100 | 720 |

La transición entre ambos tamaños se gestiona condicionalmente en el `body`:

```swift
.defaultSize(
    width: serverManager.state == .running ? 1_100 : 400,
    height: serverManager.state == .running ? 720 : 220
)
```

> **Nota:** `defaultSize` solo se aplica al abrir la ventana, no como constraint. El usuario puede redimensionarla libremente. La persistencia del tamaño se trata en Plan 10-02.

### 4. Manejo de estado de carga de WebKit

Añadir a `WebView` un `WKNavigationDelegate` que maneje:
- Error de carga (conexión rechazada si el servidor se cae): mostrar mensaje de error con botón Reload.
- Estado "cargando": `webView.isLoading` — podría mostrarse un `ProgressView` superpuesto mientras la WebView carga el HTML inicial.

```swift
class Coordinator: NSObject, WKNavigationDelegate {
    func webView(_ webView: WKWebView, didFail navigation: WKNavigation!, withError error: Error) {
        // Notificar error a la vista padre (Binding o callback)
    }
    func webView(_ webView: WKWebView, didFailProvisionalNavigation navigation: WKNavigation!, withError error: Error) {
        // Misma gestión
    }
}
```

Usar `makeCoordinator()` y pasar `Coordinator` como `navigationDelegate` en `makeNSView`.

## Notas

- La UI web existente en `static/` no requiere ningún cambio para Phase 10. Es UI agnóstica de host.
- Si la WebView intenta abrir links externos (`target="_blank"`), añadir `decidePolicyFor` en el delegate para abrirlos en el navegador del sistema con `NSWorkspace.shared.open(url)`.
- `serverPort` es la única variable de `ServerManager` que necesita la WebView — se puede pasar como `let port: Int` sin exponer `serverManager` completo.
