# Plan 11-03 — Notificaciones macOS al terminar traducción
## Phase: 11 — Translation Features & Native UI

**Estado:** PLANIFICADO  
**Objetivo:** Notificar al usuario via `UserNotifications` cuando una traducción termina, especialmente útil cuando la ventana está en segundo plano o minimizada.

---

## Contexto

- La UI web ya tiene feedback inline (spinner, resultado en el textarea de output). La notificación es complementaria, para cuando el usuario está en otra app.
- El backend devuelve la traducción síncrona en `/api/translate` y `/api/translate/file`. El SSE de batch está diferido a v3.1.
- La notificación debe dispararse desde el frontend JS, no desde Swift directamente, porque Swift no sabe cuándo termina una traducción — eso lo sabe el JS del frontend.
- El puente JS→Swift se hace via `WKScriptMessageHandler` (mensaje de la WebView a Swift).

---

## Tareas

### 1. `WKScriptMessageHandler` en `WebView.swift`

Registrar un message handler llamado `"translationDone"` que Swift escucha:

```swift
// En makeNSView(context:), añadir al WKWebViewConfiguration:
config.userContentController.add(context.coordinator, name: "translationDone")
```

```swift
// En Coordinator, conformar a WKScriptMessageHandler:
extension WebView.Coordinator: WKScriptMessageHandler {
    func userContentController(
        _ userContentController: WKUserContentController,
        didReceive message: WKScriptMessage
    ) {
        guard message.name == "translationDone",
              let body = message.body as? [String: Any] else { return }
        let filename = body["filename"] as? String ?? "traducción"
        let langs    = body["langs"]    as? String ?? ""
        NotificationManager.shared.sendTranslationDone(filename: filename, langs: langs)
    }
}
```

### 2. `NotificationManager.swift` — wrapper de `UNUserNotificationCenter`

```swift
// NotificationManager.swift
import Foundation
import UserNotifications

@MainActor
final class NotificationManager {
    static let shared = NotificationManager()

    func requestPermission() {
        UNUserNotificationCenter.current().requestAuthorization(options: [.alert, .sound]) { _, _ in }
    }

    func sendTranslationDone(filename: String, langs: String) {
        let content = UNMutableNotificationContent()
        content.title = "Traducción completada"
        content.body  = langs.isEmpty ? filename : "\(filename) → \(langs)"
        content.sound = .default
        let req = UNNotificationRequest(
            identifier: UUID().uuidString,
            content: content,
            trigger: nil  // Inmediata
        )
        UNUserNotificationCenter.current().add(req)
    }
}
```

Llamar `NotificationManager.shared.requestPermission()` en `MDTranslatorApp.body` o en `AppDelegate.applicationDidFinishLaunching`.

### 3. Inyectar JS en la WebView para disparar el mensaje

Usar `WKUserScript` para inyectar código JS que intercepte las respuestas de traducción del frontend. El frontend llama a la API y muestra el resultado — hay que interceptar ese momento.

Opción A — `WKUserScript` inyectado al cargar la página:

```swift
// En makeNSView, antes de cargar la URL:
let script = WKUserScript(source: """
window.__notifyTranslationDone = function(filename, langs) {
    if (window.webkit && window.webkit.messageHandlers.translationDone) {
        window.webkit.messageHandlers.translationDone.postMessage({
            filename: filename || 'documento',
            langs: langs || ''
        });
    }
};
""", injectionTime: .atDocumentEnd, forMainFrameOnly: true)
config.userContentController.addUserScript(script)
```

Opción B — modificar `static/js/app.js` para llamar a `window.__notifyTranslationDone(...)` tras recibir la respuesta de la API.

**Preferida: Opción B** — es más limpia y explícita. Añadir una llamada en el handler de éxito de `/api/translate` y `/api/translate/file` en `static/js/`:

```javascript
// En el callback de éxito de la traducción:
if (window.__notifyTranslationDone) {
    window.__notifyTranslationDone(filename, targetLangs.join(', '));
}
```

La Opción A (WKUserScript) es el fallback si no se quiere tocar el JS del frontend.

## Criterios de aceptación

- [ ] Primera ejecución: macOS pide permiso para notificaciones.
- [ ] Traducción completada con la app en primer plano → notificación visible en Centro de Notificaciones.
- [ ] Traducción completada con la app en segundo plano → banner de notificación en esquina superior derecha.
- [ ] El cuerpo de la notificación incluye nombre de archivo y idioma(s) destino.
