# Plan 11-02 — Drag & drop de archivos al icono del Dock
## Phase: 11 — Translation Features & Native UI

**Estado:** PLANIFICADO  
**Objetivo:** Permitir arrastrar archivos `.md` al icono del Dock o a la ventana de la app para cargarlos directamente en el editor web.

---

## Contexto

- macOS entrega los archivos arrastrados al Dock via `application(_:open:)` en `NSApplicationDelegate`.
- La UI web ya tiene zonas de drop internas (`#drop-zone`, `#drop-zone-batch`) para drag-and-drop dentro del navegador, pero no recibe archivos del sistema de archivos a través del Dock.
- La WebView puede recibir el contenido vía la misma notificación `WebView.openMarkdownNotification` ya implementada.
- Sin App Sandbox no hay restricciones de acceso a archivos arbitrarios del usuario.

---

## Tareas

### 1. Implementar `application(_:open:)` en `AppDelegate.swift`

```swift
// AppDelegate.swift — añadir:
func application(_ application: NSApplication, open urls: [URL]) {
    // Solo procesar archivos .md; ignorar el resto silenciosamente.
    let markdownURLs = urls.filter { $0.pathExtension.lowercased() == "md" }
    guard let first = markdownURLs.first else { return }

    do {
        let content = try String(contentsOf: first, encoding: .utf8)
        NotificationCenter.default.post(
            name: WebView.openMarkdownNotification,
            object: content
        )
    } catch {
        let alert = NSAlert()
        alert.messageText = "No se pudo abrir el archivo"
        alert.informativeText = "\(first.lastPathComponent): \(error.localizedDescription)"
        alert.alertStyle = .warning
        alert.runModal()
    }
}
```

> Si el usuario arrastra varios archivos, solo se carga el primero (la UI web es single-file en el tab Editor). Los archivos adicionales podrían enviarse al tab Lote en una iteración futura.

### 2. Registrar tipos de documento en `Info.plist`

Para que macOS sepa que la app acepta `.md` en el Dock, hay que declarar el tipo de documento en el target:

En Xcode → Target → Info → Document Types → `+`:
```text
Name:        Markdown Document  
Types:       net.daringfireball.markdown  
Extensions:  md  
Role:        Viewer  
```

Alternativa vía `Info.plist` directo:
```xml
<key>CFBundleDocumentTypes</key>
<array>
    <dict>
        <key>CFBundleTypeName</key>
        <string>Markdown Document</string>
        <key>CFBundleTypeExtensions</key>
        <array><string>md</string></array>
        <key>CFBundleTypeRole</key>
        <string>Viewer</string>
        <key>LSItemContentTypes</key>
        <array><string>net.daringfireball.markdown</string></array>
    </dict>
</array>
```

> Sin esta declaración, macOS no ofrece la app como destino al arrastrar `.md` en Finder ni en el Dock.

### 3. Drag sobre la ventana (opcional, si la WebView no lo absorbe)

La WKWebView absorbe todos los eventos de drag por defecto. Si se quiere drop sobre la ventana nativa (no la WebView), habría que subclasear `WKWebView` y sobreescribir `performDragOperation`. Por ahora, el flujo por el Dock + ⌘O cubre el caso de uso.

## Criterios de aceptación

- [ ] Arrastrar un `.md` al icono del Dock → se carga en el editor web.
- [ ] Arrastrar un `.md` al icono del Dock con la app cerrada → la app abre y carga el archivo.
- [ ] La app aparece como opción en "Abrir con…" para archivos `.md` en Finder.
- [ ] Archivos no-.md ignorados silenciosamente.
