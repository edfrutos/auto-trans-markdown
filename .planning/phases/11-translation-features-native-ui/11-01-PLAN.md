# Plan 11-01 — Correcciones de integración WebView↔JS
## Phase: 11 — Translation Features & Native UI

**Estado:** PLANIFICADO  
**Objetivo:** Corregir la integración JavaScript entre la capa Swift y la UI web: el id real del textarea editor es `#input-md` (no `#editor`), y el reinicio del servidor cuando se cambian las API keys en `SettingsView`.

---

## Contexto

Del análisis de `static/index.html`:
- Textarea del editor: `id="input-md"` (tab Editor)
- Drop zones: `#drop-zone` (archivo único) y `#drop-zone-batch` (lote)
- El frontend ya gestiona drag-and-drop interno vía `setupDropZone()` en `static/js/`

Del pitfall documentado en STATE.md:
- `WebView.swift` usa `getElementById('editor')` → debe ser `getElementById('input-md')`.
- `settingsSaved` notification debe reiniciar el servidor si ya estaba corriendo.

---

## Tareas

### 1. Corregir el selector JS en `WebView.swift`

```swift
// ANTES (incorrecto):
let js = """
(function() {
    const ta = document.getElementById('editor') || document.querySelector('textarea');
    ...
})();
"""

// DESPUÉS (correcto):
let js = """
(function() {
    // Activar tab Editor primero (por si está en otra pestaña)
    const tabEditor = document.getElementById('tab-editor');
    if (tabEditor) tabEditor.click();

    const ta = document.getElementById('input-md');
    if (ta) {
        ta.value = `\(escaped)`;
        ta.dispatchEvent(new Event('input', { bubbles: true }));
        ta.focus();
    }
})();
"""
```

El `tabEditor.click()` antes de inyectar garantiza que el usuario ve el panel editor, no el de archivo o lote.

### 2. Reinicio del servidor al guardar keys desde `SettingsView`

Cuando el usuario cambia keys en mitad de una sesión (servidor ya corriendo), el subprocess uvicorn sigue usando las keys anteriores — hay que reiniciarlo.

En `MDTranslatorApp.swift`, escuchar `settingsSaved` cuando el servidor está running y reiniciar:

```swift
.onReceive(NotificationCenter.default.publisher(for: .settingsSaved)) { _ in
    guard serverManager.state == .running else { return }
    // Reiniciar el servidor con las nuevas keys
    Task {
        serverManager.stop()
        // Esperar a que el proceso termine antes de arrancar de nuevo
        try? await Task.sleep(nanoseconds: 1_000_000_000)
        await serverManager.start()
    }
}
```

> **Nota:** El estado pasará por `.idle`/`.starting` momentáneamente → `SplashView` reaparecerá brevemente. Es comportamiento correcto e intencionado: el usuario ve que el servidor se está reiniciando.

### 3. Eliminar `ContentView.swift` del target Xcode

`ContentView.swift` es boilerplate generado por Xcode — no se usa en ningún lugar. Eliminarlo del target (no del disco, o directamente borrar el fichero) evita confusión futura.

Acción manual en Xcode: seleccionar `ContentView.swift` → Delete → "Move to Trash".
